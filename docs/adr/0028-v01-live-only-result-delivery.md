# ADR 0028：V0.1 只实时转发结果

## 状态

Superseded，2026-07-19。结果实时转发的部分仍有效；“不保存历史与结果”由 ADR 0030 取代。

## 背景

保存最终答案、任务历史、完成通知和重新打开后的结果恢复会引入结果数据库、保留策略、隐私设置和历史 UI。用户希望先验证浏览器到 Claude 的实时闭环，等后续 ACP 阶段再设计持久会话。

## 决策

1. V0.1 只把 Claude Code `stream-json` 产生的运行事件和文本增量实时转发给当前浏览器界面；
2. daemon 不把回复正文或最终答案写入 SQLite、文件或通知；
   Claude Code 自己按照原生默认行为保存会话不属于 Ferry 结果持久化，Ferry 不传 `--no-session-persistence`，也不读取或依赖其会话记录；
3. V0.1 不提供任务历史、结果列表、完成通知、角标提醒、重新打开查看或结果重放；
4. 最终结果成功送达当前界面后即视为本次交接完成；界面断开期间的输出不保证补发；
5. 运行所需的临时 Artifact 仍按 ADR 0011 管理，这不等于保存 Agent 结果；
6. ACP 阶段重新设计会话事件持久化、重连游标、结果保留和通知，不要求兼容 V0.1 的临时流；
7. 浏览器界面关闭或连接断开不取消仍在运行的 Claude 任务；daemon 必须继续读取 stdout/stderr，避免子进程因管道背压阻塞，但没有实时订阅者时直接丢弃回复内容；
8. V0.1 不支持重新连接到运行中任务、补发遗漏片段或在完成后取回结果；Side Panel、attach 和可靠重连延期到 ACP 阶段。

## 后果

- 首版无需设计结果数据库和历史 UI；
- 用户必须保持实时界面连接才能看到完整答案；
- 浏览器或扩展断开会使后续结果永久丢失，但长任务仍持续运行到完成、用户显式取消或超时；
- V0.1 接受 popup 生命周期短带来的输出丢失，以避免提前实现 ACP 阶段的 attach、历史和持久 UI。
