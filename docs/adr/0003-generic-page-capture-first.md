# ADR 0003：V0.1 优先通用整页捕获

## 状态

已被 ADR 0004 取代，2026-07-14。

## 背景

不同站点对正文、线程、回复、推荐内容和动态加载采用不同的页面结构。若 V0.1 就为论文站点、Twitter/X 和各类技术文档建立专用语义解析器，会扩大首版范围，并让交付进度受站点变化影响。

当前首要目标是先跑通从浏览器到 Agent 的可靠交接闭环。

## 决策

V0.1 采用通用整页捕获策略：尽可能提取浏览器页面中可读取的全部内容，不在捕获阶段区分主内容、回复、推荐内容或其他站点特有区域。

站点专用提取器、内容范围选择和语义降噪作为后续增强，不进入首个纵向闭环。

## 备选方案

### 首版实现站点专用提取器

暂不采用。它能提高内容质量，但会让首版依赖多个站点的私有 DOM 结构和持续维护。

### 只捕获用户选区

不作为默认策略。它要求用户手工选择内容，无法满足一键交接整篇论文或长文档的目标。

## 后果

- 首版捕获内容可能包含导航、回复、推荐信息和其他噪声；
- Artifact 必须标明捕获方式和可能的不完整性，不能暗示已经完成语义正文提取；
- Agent 的默认 Prompt 可以要求先识别主要内容，再进行分析；
- Content Plane 应保留未来增加站点适配器的扩展点，但不能提前实现复杂的站点规则系统；
- 需要另行明确动态页面中“可读取的全部内容”是否包括尚未加载的内容。
