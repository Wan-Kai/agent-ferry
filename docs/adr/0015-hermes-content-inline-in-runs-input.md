# ADR 0015：云端 Hermes 正文直接放入 Runs API input

## 状态

已接受，2026-07-15。

## 背景

本地 Claude 可以读取 daemon 创建的临时 Artifact，但云端 Hermes 无法访问 macOS 临时目录。Hermes API Server 当前不支持上传普通文档，提前设计分块导入和服务器端组装协议会扩大首版范围，而且尚无真实数据证明这一复杂度必要。

## 决策

V0.1 将最终 Prompt、来源信息和捕获到的完整 Markdown 组成一次 Runs API 请求，直接放入 `input`：

1. 不为云端 Hermes 创建或传输本地文件路径；
2. 不通过 SSH/SFTP 把正文写入服务器；
3. 不静默截断正文；
4. 若请求大小、服务端配置或模型上下文限制导致失败，明确报告失败原因，并保留本地捕获结果用于重试；
5. 分块导入、服务器端组装和大文档上传协议在出现真实失败样本后再设计。

Hermes 收到正文后，仍由其自身决定是否保存到服务器文件、保存位置和召回方式。

## 后果

- 云端交接链路保持为一次 HTTP 请求和后续 SSE 事件；
- 常规网页和论文无需额外的文件传输协议；
- 超大内容可能无法完成首版交接，但系统必须显式失败，不能生成看似成功的截断结果；
- 需要记录原始字符数、请求字节数和服务端错误类型，为后续是否引入分块提供依据。
