# ADR 0022：宿主 Agent 可执行文件的解析与绑定

## 状态

已接受，2026-07-15。

## 背景

一台机器可能通过 Homebrew、官方安装器、版本管理器或用户目录同时安装多个 Claude Code。仅依赖 daemon 启动时的 `PATH` 会受到启动方式和环境差异影响，也可能在升级后悄然切换到另一个版本。

## 决策

1. doctor 使用受信任的探测规则收集候选可执行文件，并将候选规范化为绝对路径；
2. 只发现一个兼容候选时可以自动选择，并持久化其绝对路径和已验证版本；
3. 发现多个兼容候选时不得自行猜测，目标进入 `needs_selection`，CLI 展示每个候选的绝对路径和版本；
4. 用户通过 `aferry agent enable claude-code --command <absolute-path>` 明确选择；
5. `--command` 只接受单个绝对可执行文件路径，不接受参数、shell 表达式、alias、管道或命令替换；
6. 每次启动 Agent 会话前检查文件仍存在、可执行且版本仍兼容；路径失效或版本不兼容时停止启动并提示重新运行 doctor；
7. Ferry 不因 `PATH` 顺序变化静默替换已经绑定的宿主可执行文件。

## 后果

- 首次发现多个安装时需要用户多做一次选择；
- daemon 不依赖交互 shell 初始化文件，也不会在后台换用另一个 Claude Code；
- Agent 配置需要保存宿主绝对路径、已验证版本和最后检查时间；
- 浏览器只显示 `needs_selection` 和修复指引，不提供任意命令输入框。
