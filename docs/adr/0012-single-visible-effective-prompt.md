# ADR 0012：交接前只编辑一个完全可见的最终 Prompt

## 状态

已接受，2026-07-15。

## 背景

扩展需要同时支持内置默认 Prompt、用户配置的模板和本次交接的临时修改。如果将这些输入在发送时隐式拼接，用户难以确认 Agent 实际收到的指令，也容易产生模板优先级和覆盖规则上的歧义。

## 决策

扩展在交接界面只提供一个“最终 Prompt”编辑框：

1. 选择模板后，将模板解析并填入最终 Prompt；
2. 模板变量在编辑框中解析为实际值，包括页面标题、来源 URL 和本地临时 Artifact 路径等当前目标可用信息；
3. 用户可以在发送前直接编辑最终文本；
4. 本次编辑只影响当前 Handoff，不反向覆盖已保存模板；
5. 未选择模板时，编辑框展示内置默认 Prompt；
6. 系统不会在编辑框内容之外追加用户不可见的任务指令。

传输协议需要附带来源、目标和 Artifact 等结构化数据，但这些数据不能被用来偷偷改变用户任务。为了让本地 Claude 知道如何读取 Artifact，面向 Agent 的读取说明也必须出现在最终 Prompt 中。

## 后果

- 用户发送前看到的文本就是 Agent 收到的任务说明；
- V0.1 不需要定义“模板 + 临时目标”的隐式拼接优先级；
- 切换模板会替换编辑框内容，界面需要在用户已有未保存编辑时避免误覆盖；
- 模板管理与单次 Prompt 编辑是两个独立操作；
- 不同目标不支持的变量必须在交接前报错，不能把未解析占位符静默发送给 Agent。
