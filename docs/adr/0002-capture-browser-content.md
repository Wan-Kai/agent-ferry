# ADR 0002：交接浏览器内容快照而非仅交接 URL

## 状态

已接受，2026-07-14。

## 背景

用户希望把浏览器中正在阅读的论文、Twitter/X 帖子或技术文档交给本地 Claude Code 或远程 Hermes。目标 Agent 可能没有浏览器登录态，也可能无法访问付费内容、动态页面或已发生变化的页面。

如果 Agent Ferry 只发送 URL，交接结果将依赖 Agent 的网络环境和重新抓取能力，无法保证 Agent 实际看到用户希望分析的内容。

## 决策

Agent Ferry 必须从用户当前浏览器会话中提取用户能够访问的实际内容，并将其作为交接内容。页面 URL、标题和抓取时间作为来源元数据保留，但 URL 不能代替内容本身。

这一原则适用于普通 HTML 文档、论文 PDF 和 Twitter/X 等动态页面。不同页面类型可以使用不同提取器，具体支持范围按里程碑明确。

## 备选方案

### 只发送 URL

不采用。远程 Agent 不具备浏览器登录态时会丢失内容，也无法形成稳定、可复查的交接记录。

### 由 Agent 自行抓取，浏览器内容作为可选补充

不作为默认路径。Agent 可以根据任务需要再次联网核查，但首次交接必须包含浏览器捕获的内容。

## 后果

- Chrome 扩展需要读取页面内容，而不只是读取标签页 URL；
- PDF、动态页面和普通文章可能需要不同的提取策略；
- 捕获结果需要标明来源、时间和可能的不完整性；
- 浏览器登录凭据、Cookie 和其他认证信息不能随内容传给 Agent；
- 页面中的提示注入文本仍然是不可信内容，不能变成 Ferry 的控制指令。
