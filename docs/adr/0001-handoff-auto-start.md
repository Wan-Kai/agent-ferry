# ADR 0001：交接后自动开始 Agent 对话

## 状态

已接受，2026-07-14。

## 背景

首要用户场景是把浏览器中正在阅读的论文、Twitter/X 帖子或技术文档交给本地 Claude Code 或远程 Hermes 分析。用户希望消除复制内容、切换终端、进入固定目录、启动 CLI 和手工发送首条消息的重复操作。

仅打开 Agent CLI 或预填输入仍要求用户到终端完成二次确认，不能完成端到端交接。

## 决策

用户在扩展中点击交接按钮，即代表授权 Agent Ferry 执行本次交接。系统在目标工作区启动 Agent 后，必须自动提交首条消息并让 Agent 立即开始工作。

内容写入、Agent 启动和首条消息提交应作为可区分的阶段报告结果，不能把“CLI 已打开”视为交接成功。

## 备选方案

### 只打开 CLI

不采用。它保留了手工发送消息的步骤，没有解决完整用户问题。

### 预填消息并等待用户确认

不作为 V0.1 默认行为。未来可以增加预览或草稿模式，但不能改变默认交接语义。

## 后果

- 扩展中的交接按钮是一次有实际执行效果的操作，界面需要清楚展示目标 Agent、工作区和将要执行的目标；
- Agent 启动适配器必须支持可靠提交初始消息，不能只支持打开可见终端；
- 网页内容属于不可信数据，必须与 Ferry 的控制指令隔离；
- 需要分别呈现内容写入、进程启动和消息提交阶段的错误；
- 自动开始仅授权当前交接，不代表默认批准 Agent 后续提出的危险权限请求。
