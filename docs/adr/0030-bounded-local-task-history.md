# ADR 0030：保存有界的本地任务历史

## 状态

Accepted，2026-07-19。取代 ADR 0028 和 ADR 0029 中“Ferry 不保存任务历史或回复”的结论。

## 背景

Popup 生命周期很短，长任务在界面关闭后继续运行。只实时转发会让用户无法确认后台任务是否完成，也无法取回已经生成的结果。当前实现已经为本机与远程目标提供统一任务历史、详情和删除能力。

## 决策

1. daemon 在当前用户私有目录按 task id 保存 JSON 历史；
2. 历史包含 Prompt、来源摘要、目标快照、归一化状态、最终或截断输出、非敏感错误和有界事件，不保存捕获正文或凭据；
3. 输出最多保存 768 KiB，事件最多 500 条，记录最多 500 个；达到上限时只清理终态旧记录；
4. 历史目录权限为 `0700`，记录以 `0600` 临时文件写入后原子替换；
5. daemon 重启时把遗留的运行中任务标记为 `interrupted`，不伪造任务恢复；
6. 运行中记录不能删除，终态记录允许用户显式删除；
7. Chrome Principal 只获得历史读取和删除 capability，历史不得暴露 Agent 凭据或 Connector Secret。

## 后果

- 用户关闭并重新打开 Popup 后可以看到任务状态与有界输出；
- Ferry 承担本地历史的隐私、容量、兼容和删除责任；
- 历史不等于可恢复的 Agent Session，也不保证 daemon 重启后继续执行任务；
- Remote Hermes 自己的记忆和文件仍由 Hermes 管理，本地历史只是 Ferry 的任务可观察性记录。

## 被取代决策

- ADR 0028 的实时流与“关闭 UI 不取消任务”仍保留，但“不保存历史与结果”被本 ADR 取代；
- ADR 0029 的 Hermes 所有权边界仍保留，但 Hermes 任务的 Ferry 本地观察历史由本 ADR 统一定义。
