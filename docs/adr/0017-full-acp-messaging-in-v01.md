# ADR 0017：V0.1 完成 ACP 消息会话闭环

## 状态

已被 ADR 0026 取代，2026-07-15。完整 ACP 消息闭环延期到后续版本。

## 背景

本地 Claude 默认通过 ACP Managed Session 运行，不再打开终端。如果首版只提交第一条消息并显示最终结果，用户无法继续对话，也无法充分利用 ACP 已提供的流式内容、工具调用、权限请求、取消和会话能力。

ACP 是公开协议，社区已有客户端 SDK、编辑器集成和会话 UI 可供参考。把完整消息连接延后，反而会让首版先实现一套短命的一次性任务接口，再在后续重构为真正的会话模型。

## 决策

V0.1 完成本地 Claude 的 ACP 消息会话闭环：

1. `agentferryd` 作为 ACP Client 启动并持有 Claude ACP adapter；
2. 支持创建会话、提交首条 Prompt 和继续向同一 Session 发送消息；
3. 支持流式 Agent 消息、工具调用事件、权限请求、错误和完成状态；
4. 支持取消当前 Turn；
5. 扩展关闭后 daemon 继续持有会话，重新打开扩展可以恢复任务详情和继续发送；
6. 浏览器扩展提供消息记录、输入框、任务状态和权限审批界面；
7. 优先复用或适配社区 ACP SDK 和公开实现，不自行发明协议变体。

会话恢复的精确语义以 Claude ACP adapter 实际声明的能力为准。若 adapter 不支持跨进程恢复，daemon 重启后必须明确标记会话不可继续，不能伪造已恢复状态。

## 后果

- ACP Client、会话存储和事件转发成为 V0.1 核心，而不是 V0.2 增强项；
- Handoff 创建后会关联一个可持续交互的 Agent Session；
- 扩展 UI 从一次性交接表单扩展为任务详情与轻量会话界面；
- daemon 需要处理背压、事件序号、断线、进程退出和并发发送约束；
- 仍不实现 Agent Loop，所有推理和工具行为由 Claude 保持所有权。
