# ADR 0016：一次交接只选择一个目标

## 状态

已接受，2026-07-15。

## 背景

本地 Claude 与云端 Hermes 的内容传输、会话协议、权限请求和结果语义不同。允许一次 Handoff 同时发送到多个目标，会立即引入并发启动、部分成功、重复副作用、取消传播和结果聚合等额外问题。

## 决策

V0.1 中一个 Handoff 只包含一个 Target：本地 Claude 或云端 Hermes。

用户需要把同一内容交给两个 Agent 时，分别发起两次 Handoff。系统可以复用已经捕获的页面内容，但两次任务拥有独立的 ID、Prompt、状态、权限请求和结果。

V0.1 不提供多目标勾选、广播、Agent 间协作或结果聚合。

## 后果

- `Handoff.target` 是单个带类型判别的值，不是列表；
- 每个任务只有一个 Controller 和一条状态机；
- 重试只影响当前目标，不会重复触发其他 Agent；
- 未来若引入广播，应在 Handoff 之上建立显式的 Batch/Workflow，而不是改变单任务语义。
